12. 数据存储

12.1. Mirror 镜像同步

12.2. RAID

12.2.1. 缓存服务器

全部采用RAID 0

一旦出现问题，立即将其从集群中踢出去，带节点故障排除后，恢复它的功能。

12.2.2. Web 服务器

采用RAID 1

服务器仅仅存放脚本程序，数据建议放在外挂存储上。

12.2.3. 数据库

主服务器：建议采用 RAID 10

数据库节点：建议采用 RAID 10

数据库应尽量避免使用RAID 5，RAID 5在做校验过程时，效率会很低。

数据库节点一旦出现问题，立即从集群中撤出，排除故障后，在回复使用。

12.2.4. 数据备份

数据备份服务器建议采用RAID 5

RAID 5 阵列容量计算公式 ：

可用容量 =（n-1）/n的总磁盘容量（n为磁盘数）

12.3. File System 文件系统

我个人推荐使用 xfs 或 reiserfs

12.4. Network storage 网络存储

12.4.1. Distributed File System(DFS)

RAID 0提高吞吐能力是有限的，IO也会有瓶颈，NAS吞吐能力一样有限，SAN价格不菲。

DFS是一个不错的选择

12.4.2. Network-attached storage

NAS 说白了就是一个嵌入式电脑，经过精简内核的Linux,通过samba,nfs,WebDav,ftp...等等方式实现共享存储

如果你有兴趣，可以DIY一个NAS，使用Openfiler
