2. XHTML+DIV+CSS

為何使用表格排版是不明智的選擇？為什麼要選擇DIV+CSS？

首選我來說說表格排版，表格排版也是有好處的，一是排版速度快，二是兼容性比CSS好。做為一般的小網站還是比較適合的，如果在大型網站使用表格就不太合適。 表格必須定義很多屬性如width="100%" border="0" cellpadding="0" cellspacing="0"，並且有時候tr標籤顯得多餘。

上面例子我們可以看到div與table相比所使用的標籤更少，無形中給網站減了肥。

CSS 的class,id 名稱定義規範：

  1. 一定要簡單，可讀例如header，footer

  2. 對於在頁面中不重複，自始至終只出現一次可定義為id，例如id="header"，id="footer"

  3. 對於在頁面中經常重複出現的，可定義為class，例如id="article_block"，id="news_block"

注意

不要使用HTML屬性，儘量使用css。 herf,src,class,id等屬性除外。

下面是一個例子

			
<font color="red" size="12" face="Arial, Helvetica, sans-serif" >Hello workd</font>

你應該使用CSS實現,如果能使用CSS實現儘量不要多用一條HTML和屬性。

			
<style type="text/css">
.hello{
	color:red;
	font-size:12px;
	font-family:Arial, Helvetica, sans-serif;
}
</style>
<div id="hello">Hello workd</font>
