1.2. Administration

进入Admin初始化Trac系统。

1.2.1. General

安装后首先分配权限

我习惯于为每个部门创建一个组, 然后创建用户隶属于部门组

过程 1.1. Permissions 设置

  1. admin 组

    trac-admin trac permission add admin TRAC_ADMIN

  2. 开发组

    创建用户隶属于Development组

    trac-admin trac permission add Development BROWSER_VIEW  CHANGESET_VIEW  FILE_VIEW

  3. 运维组

    trac-admin trac permission add Operations TICKET_ADMIN  WIKI_ADMIN

  4. 产品组

    trac-admin trac permission add Production TICKET_ADMIN  WIKI_ADMIN

创建用户

trac-admin trac permission add mgmt admin
trac-admin trac permission add neo Development
trac-admin trac permission add ken Operations
trac-admin trac permission add jam Production
trac-admin trac permission list

Repositories

关于版本库项目目录，我习惯与使用该项目对应的域名作为项目目录

/example.com

1.2.3.1. Subversion

默认支持 Subversion, 创建一个仓库记得不要忘记创建下面三个目录 1.branches, 2.tags, 3.trunk

				
trunk			主干
branches		在下面再创建两个目录development,testing
tags			当项目Release 后会在此处打一个标记

1.2.3.2. Git

Git 不需要这三个目录，我习惯上会创建几个分支

master			主干
development		开发分支
testing			测试分支
